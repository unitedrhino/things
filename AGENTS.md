# AGENTS.md

此文件为支持读取 `AGENTS.md` 的 AI 开发工具提供在此代码库中工作的根级行为准则。

> **同步维护要求**
> - 本目录中的 `AGENTS.md` 与 `CLAUDE.md` 是配对文档，更新任意一份时必须同步更新另一份

## 核心行为准则

### 1. 先澄清再动手

- 不要默认猜测需求、上下文、运行环境或用户意图
- 如果一个请求存在多种合理解释，先明确分歧点，再执行
- 修改前先阅读对应作用域文档

### 2. 简单优先

- 只做本次请求直接需要的内容，不扩功能，不补"顺手优化"
- 不为了未来可能用到的场景提前抽象、配置化、通用化

### 3. 手术式修改

- 只改与当前任务直接相关的文件、段落和逻辑
- 不顺手重构无关代码，不顺手修格式
- 只清理由本次改动直接产生的废弃内容

### 4. 目标驱动验证

- 动手前先定义"完成"的判断标准
- 能测试就测试，能检查就检查

### 5. 确定路径直接执行

- 对于目标明确、实现路径已确定、风险可控的任务，直接持续执行直到完成
- 涉及删除数据库、清空数据库、执行不可逆数据删除时，必须先获得用户明确确认

### 6. 并行化与效率优先

- **能并行则并行**：多个独立子任务必须使用 subagent 并行执行
- **设置合理超时**：启动 subagent 时必须设置合理的超时时间

## 跨工具 AI 上下文分层

- `AGENTS.md`：仓库根级最小规则集
- `CLAUDE.md`：详细规范、专项流程、踩坑记录

## 根级工作规则

### 1. 知识更新必须落盘

- 通用规范 → 同步更新 `AGENTS.md` + `CLAUDE.md`

### 2. 原子提交与测试驱动

- **测试通过后立即提交**
- **不推送远程**：默认只提交到本地
- **不动无关代码**
- **提交粒度**：一次提交只做一件事

### 3. Git 工作规则

- **提交前必须先同步远端**：`git pull --no-rebase`
- pull 策略：**只允许 merge，禁止 rebase**
- `git commit` 提交说明必须使用中文
- 提交说明第一行用简洁中文标题，正文多行中文补充背景、改动、影响范围

### 4. 代码注释与文档规范

- 新增或修改的函数、方法、结构体、接口、常量、变量，必须在定义处用中文注释说明用途
- 文件头部必须包含中文文件说明
- 注释只写"为什么"和"做什么"
- 后端 Go 公开函数遵循 Go Doc 规范，注释以函数名开头，后接中文说明

### 5. 环境操作前置规范

- 任何涉及远程环境的操作之前，必须先阅读 `deploy/environments/` 下对应环境的说明文档
- 禁止仅凭记忆或历史经验直接操作环境
